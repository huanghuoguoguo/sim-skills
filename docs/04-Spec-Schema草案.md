# Spec.md 规范

## 1. 定位

`spec.md` 是上游的核心产物，也是下游的核心输入。

- **上游产出**：Agent 从参考文件中提取，用户精校
- **下游消费**：Agent 读取后指导检查，Python 工具接受 Agent 翻译的指令

不再需要 `spec.json`。规则以自然语言编写，Agent 和用户都能直接读。

## 2. 设计原则

- **人可读**：用户能直接审核和修改
- **结构清晰**：Agent 能可靠地逐条解析
- **可执行**：每条规则足够具体，能被翻译为检查动作
- **完备**：覆盖常见格式要素，遗漏显式标注

## 3. 推荐结构

```markdown
# 某大学本科毕业论文格式规范

## 来源
- 模板文件：template.docx
- 格式说明：format-guide.docx

## 页面设置
- 纸张：A4
- 页边距：上 2.54cm，下 2.54cm，左 3.0cm，右 2.5cm

## 正文
- 中文字体：宋体
- 西文字体：Times New Roman
- 字号：小四（12pt）
- 行距：1.5 倍行距
- 首行缩进：2 字符
- 段前距：0pt
- 段后距：0pt
- 对齐方式：两端对齐

## 标题
### 一级标题
- 中文字体：黑体
- 字号：三号（16pt）
- 对齐：居中
- 段前距：24pt
- 段后距：18pt

### 二级标题
...

## 图题
...

## 表题
...

## 摘要
- 中文摘要：...
- 英文摘要：...

## 参考文献
...

## 待确认项
- [ ] 页眉页脚具体内容待确认
- [ ] 附录格式待确认
```

## 4. 编写要求

### 每条规则必须具体可执行

好：`中文字体：宋体，字号：小四（12pt）`
差：`字体要规范`、`格式要合适`

### 属性表述统一

- 字号统一写 `小四（12pt）` 格式，中文名 + pt 值
- 行距写 `1.5 倍行距` 或 `固定值 20pt`
- 缩进写 `2 字符` 或 `0.74cm`
- 边距统一用 cm
- 段距统一用 pt

### 不确定项显式标注

提取过程中无法确定的规则，放入"待确认项"，不要猜。

## 5. 质量评估标准（evaluate-spec 用）

Agent 自评 spec.md 时检查：

1. **覆盖性**：是否包含页面设置、正文、标题、图题、表题等常见要素
2. **具体性**：每条规则是否有明确的属性值（不是模糊描述）
3. **一致性**：属性单位是否统一，有无矛盾
4. **可追溯**：是否标注了规则的来源（哪个文件、哪个样式）
5. **完备性**：是否标注了待确认项

---

> python3 -m sim_docs spec-check --mode conflicts spec.md

> python3 -m sim_docs spec-check --mode body-consistency --evidence evidence.json --checks checks.json
> ```
