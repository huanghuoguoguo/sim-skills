# 文档格式检查 Skills 产品设计

## 1. 设计目标

设计一个围绕"Schema 提取与格式检查"的技能集，以 `.docx` 论文场景作为第一条落地链路。

系统支持两条主流程：

1. **模板/成品 -> Schema 提取**
2. **Schema + 待检查文档 -> 合规检查报告**

系统输出不是模糊建议，而是结构化、可解释、可复核的检查结果。

## 2. 核心原则

- 程序优先：能用程序判断的，不交给模型猜
- AI 补全：只让 AI 处理非结构化规范和模糊语义
- 可解释：每条错误都必须能说明"要求值、当前值、定位、依据"
- 可沉淀：Schema 可保存和复用

## 3. 系统能力边界

### 3.1 输入

- 模板 `.docx`：学校提供的空白模板
- 成品 `.docx`：学长学姐的合格论文
- 待检查 `.docx`：用户当前需要检查的文档
- Schema `json`：已有的规则文件

### 3.2 输出

- Schema JSON：结构化规则
- 检查报告 JSON
- 检查报告 Markdown（面向用户可读）

## 4. 总体架构

```
主线一：Schema 提取
模板/成品.docx -> [inspect-word] -> Document IR -> [extract-schema] -> schema.json

主线二：格式检查
待检查.docx + schema.json -> [inspect-word] -> Document IR -> [check-format] -> report.md
```

## 5. 模块设计

### 5.1 inspect-word

职责：
- 解压并读取 `.docx` OOXML
- 提取段落、run、样式、编号、节、页边距、页眉页脚、图片、表格、题注等信息
- 输出 Document IR（中间表示）

### 5.2 extract-schema

职责：
- 从 Document IR 中提取可复用的规则
- 合并多个文档的规则（如有）
- 输出 schema JSON

### 5.3 check-format

职责：
- 读取 schema 和待检查文档的 IR
- 逐项比对规则
- 生成问题报告

## 6. 规则模型设计

规则采用"选择器 + 属性约束"的形式。

### 6.1 顶层 schema

```json
{
  "spec_id": "school-thesis-v1",
  "name": "某大学本科毕业论文格式规范",
  "scope": "undergraduate_thesis",
  "version": "1.0.0",
  "rules": []
}
```

### 6.2 selector 设计

示例：
- `body.paragraph`：正文段落
- `body.heading.level1`：一级标题
- `body.heading.level2`：二级标题
- `figure.caption`：图题
- `table.caption`：表题

### 6.3 property 设计

首批字段：
- `font_family_zh`：中文字体
- `font_family_en`：英文字体
- `font_size_pt`：字号
- `line_spacing_pt`：行距
- `first_line_indent_chars`：首行缩进
- `space_before_pt`：段前距
- `space_after_pt`：段后距
- `alignment`：对齐方式
- `page_margin_*_cm`：页边距

## 7. AI 与程序的职责分工

### AI 负责
- 生成面向用户的错误解释
- 处理模糊或不确定的规则

### 程序负责
- 读取 artifact
- 提取结构与属性事实
- 执行规则匹配
- 输出可复核结果

## 8. 用户路径

### 路径 A：提取 Schema

1. 用户提供模板.docx 和/或成品.docx
2. 调用 `inspect-word` 解析文档
3. 调用 `extract-schema` 生成 schema
4. 保存 schema.json 供后续使用

### 路径 B：检查文档

1. 用户选择已有 schema.json
2. 提供待检查.docx
3. 调用 `inspect-word` 解析文档
4. 调用 `check-format` 生成报告
5. 输出 report.md 和 report.json

## 9. 风险与应对

### 9.1 规范文本含糊

应对：
- 允许规则置信度
- 输出待确认项
- 保留源片段引用

### 9.2 Word 文档结构不规范

应对：
- 支持"按样式识别"和"按文本模式识别"双通道
- 允许对未套样式文档做启发式识别

### 9.3 页面级判断难

应对：
- MVP 阶段先做"近似定位"
- 不把真实页码、跨页题注、孤行控制纳入首版硬承诺
