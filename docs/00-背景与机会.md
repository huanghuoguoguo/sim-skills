# 文档格式检查：背景与机会

## 1. 项目定位

- 项目代号：`doc-format-skills`
- 当前场景：高校论文 `.docx` / `.dotm` 格式检查
- 目标形态：Agent（Claude Code）自主编排工具脚本，完成格式规则提取和文档检查
- 核心理念：**架构简单，实现可以复杂**

## 2. 问题定义

高校论文格式要求分散在多个来源：

- 学校规范 PDF / Word 说明文档
- 学院提供的 Word 模板（样式即元数据）
- 学长的成品论文（格式示例）
- 导师或教务的补充说明（自然语言描述）

用户面临的核心困难：

- 规范信息分散，难以归一
- 模板样式和文字说明经常不一致
- 手动逐条对照检查耗时且容易遗漏

## 3. 解决思路

不做一个黑盒工具，而是：

- 提供一组**可单步执行的工具脚本**（Python），负责文档解析、样式查询、格式检查等确定性工作
- 由 **Agent 自主规划和编排**，决定调用什么工具、以什么顺序、如何综合结果
- 用 **Skill（prompt）** 指导 Agent 的工作方向和质量标准

Agent + 工具脚本的组合，天然适合这种需要"多步取证 + 综合判断"的任务。

## 4. 两条用户路径

### 上游：提取规则

用户提供一批参考文件 → Agent 调用工具分析 → 输出 `spec.md`（自然语言格式规则）→ 用户精校

### 下游：检查文档

用户提供论文 + `spec.md` → Agent 调用工具检查 → 输出检查报告

两条路径共享同一批工具脚本，由不同的 Skill 指导 Agent 的工作方式。

## 5. 首阶段边界

支持：
- 输入格式：`.docx` / `.dotm`
- 主场景：高校论文 / 毕业设计
- 主产物：`spec.md`（规则）、检查报告（Markdown）

暂不优先：
- 自动修复文档
- PDF 作为主检查对象
- 复杂公式、文本框、图形对象的精细校验
