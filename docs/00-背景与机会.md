# 文档格式检查 Skills 项目背景与机会

## 1. 文档信息

- 项目代号：`doc-format-skills`
- 目标：为"模板/规范 -> 规则元数据 -> 产物检查 -> 错误报告"提供 Claude Code Skills 工具集
- 当前落地场景：高校论文 `.docx` 格式检查
- 交付形式：`.claude/skills` 目录下的可调用技能

## 2. 问题定义

高校论文、毕业设计报告、课程论文都存在大量格式合规要求。这些要求通常分散在：

- 学校撰写规范 PDF
- 学院提供的 Word 模板
- 历届合格样例
- 导师或教务补充说明

用户常见痛点：

- 不知道要求到底是什么，规范文本分散且含糊
- 即使知道要求，也很难逐条在 Word 中手工核对
- 学校之间规则差异大，通用模板工具无法直接复用

本项目提供两条主线能力：

1. **Schema 提取**：从模板/成品文档中自动提取结构化规则
2. **格式检查**：基于已有 schema 检查文档是否合规

## 3. 目标用户

- 本科生、硕士生、博士生
- 指导老师、教务人员
- 提供论文辅导服务的机构

## 4. 产品定位

项目定位是面向 Claude Code 用户的格式检查技能包：

```
用户上传文档 -> Claude Code Skills -> 提取 schema / 检查合规 -> 输出报告
```

核心能力：

- 模板/规范理解
- 规则元数据生成
- 文档结构与属性解析
- 规则匹配与错误定位
- 报告生成

## 5. 两条主线

### 主线一：Schema 提取

```
输入：模板.docx + 成品.docx（可选：规范文本）
输出：schema.json
```

流程：
1. 用户上传模板和/或成品文档
2. 工具解析文档，提取样式、字体、间距等属性
3. 输出结构化的 schema JSON 文件

### 主线二：格式检查

```
输入：待检查.docx + schema.json
输出：report.md + report.json
```

流程：
1. 用户选择已有 schema
2. 上传待检查文档
3. 工具逐项比对，输出问题报告

## 6. 交付形式

所有能力封装为 `.claude/skills` 目录下的技能：

- `extract-schema`：从模板/成品提取 schema
- `check-format`：基于 schema 检查文档
- `inspect-word`：解析 .docx 文件，提取结构化信息

脚本源码位于 `skills/resources/` 目录。

## 7. 范围边界

首阶段只做：

- 输入主格式：`.docx`
- 主场景：高校论文/毕业设计
- 主能力：检查与报告，不自动修复

暂不优先做：

- PDF 作为主检查对象
- 复杂公式、文本框的细粒度检查
- 完整参考文献内容正确性判断

## 8. 成功标准

- 能从真实模板中生成可用 schema
- 能检查正文、标题、摘要、图表题注等主要模块
- 输出结果可理解、可操作
- 接入新学校模板时，只需重新提取 schema
